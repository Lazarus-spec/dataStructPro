# 入门篇
##　复杂度：
方法：
1. 根据代码执行次数进行估算（先假定每行代码执行时间相同）
2. 从 CPU 的角度来看，这段代码的每一行都执行着类似的操作：读数据-运算-写数据

    T(n) = O(f(n))
    
    大 O 时间复杂度实际上并不具体表示代码真正的执行时间，**而是表示代码执行时间随数据规模增长的变化趋势**

原则：
1. 只关注循环执行次数最多的一段代码
2. 加法法则：总复杂度等于量级最大的那段代码的复杂度
3. 乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积
```
复杂度：O(n2)。

int cal(int n) {
   int ret = 0; 
   int i = 1;
   for (; i < n; ++i) {
     ret = ret + f(i);
   } 
 } 
 
 int f(int n) {
  int sum = 0;
  int i = 1;
  for (; i < n; ++i) {
    sum = sum + i;
  } 
  return sum;
 }
```
